# 示例：诊断从截图

## 用户输入
```
[上传YouTube Studio后台截图]
帮我看看这个频道怎么样
```

## AI响应流程

### Step 1: 截图识别

AI检查截图中存在哪些数据字段：

```
✅ 已识别字段：
- 订阅数：2,340
- 总播放量：189,432
- 视频数：47
- 平均观看时长：1:23 (83秒)
- 30天观看时长：12,450小时

❌ 缺失字段（截图中未显示）：
- CTR（展示点击率）
- 留存曲线
- 流量来源饼图
- 受众年龄/性别分布
```

### Step 2: 判档

有部分Analytics数据 → **B档**（有播放/时长，无CTR/留存）

### Step 3: 运行诊断流程

根据已识别字段，能诊断：
- ✅ 播放分布（基于视频数和总播放）
- ✅ 订阅转化（基于订阅数和播放量）
- ✅ 发布频率（基于视频数和频道年龄）
- ⚠️ 时长分析（有平均观看时长，但无视频时长对比）
- ❌ CTR分析（缺失）
- ❌ 留存分析（缺失）
- ❌ 流量来源分析（缺失）

---

## 关键点

1. **截图识别协议**：AI逐项检查截图中有哪些字段
2. **已识别vs缺失**：明确告诉用户"我看到了什么"和"我还缺什么"
3. **B档诚实标注**：有部分数据但不完整，置信中等
4. **引导补数据**：告诉用户下次截图应该包含哪些内容
5. **不硬编**：缺失的数据维度不猜测，直接进skipped

---

## 截图识别清单

AI收到截图时，逐项检查：

- [ ] 订阅数
- [ ] 总播放量
- [ ] 视频数
- [ ] 平均观看时长
- [ ] 30天观看时长
- [ ] CTR（展示点击率）
- [ ] 留存曲线截图
- [ ] 流量来源饼图
- [ ] 受众年龄/性别分布
- [ ] 地域分布
- [ ] 设备分布

存在的字段打勾，缺失的告诉用户"还需要补X截图"。
