# 四象限分类规则

## CTR×AVD 二维矩阵

这是诊断的核心骨架。每条视频必须先归类到象限，再给针对性建议。

| 象限 | CTR | AVD | 诊断 | 动作 |
|---|---|---|---|---|
| **爆款基因** | ≥6% | ≥15% | 门面+内容双优 | 继续剪同类，量产模板 |
| **标题超卖_开头型** | ≥6% | <10% | 标题吸引但内容留不住 | 优化前30秒hook，检查中段节奏 |
| **标题超卖_中段型** | ≥6% | 10-15% | 开头OK但中段流失 | 每3-5分钟设置re-engagement hook |
| **门面拖累** | <2.5% | ≥15% | 内容好但没人点 | 重置封面+标题 |
| **选题失败** | <2.5% | <10% | 门面+内容双差 | 选题方向不适合此频道 |
| **表现平庸** | 2.5-6% | 10-15% | 中等偏下 | 微调标题+封面 |
| **样本不足** | — | — | 展示<500 | 不给动作，等观察 |

## 象限归类算法

```
if impressions < 500:
    bucket = "样本不足"
elif CTR >= 6% and AVD >= 15%:
    bucket = "爆款基因"
elif CTR >= 6% and AVD < 10%:
    bucket = "标题超卖_开头型"
elif CTR >= 6% and 10% <= AVD < 15%:
    bucket = "标题超卖_中段型"
elif CTR < 2.5% and AVD >= 15%:
    bucket = "门面拖累"
elif CTR < 2.5% and AVD < 10%:
    bucket = "选题失败"
else:
    bucket = "表现平庸"
```

## 爆款判断公式

```
爆点得分 = CTR(%) × 完播率(%) × 观看时长(min) / 100
> 30   → 爆款，同类48h内加倍
15-30  → 潜力，继续观察
< 15   → 一般，优化或放弃
```

## 48小时诊断表

| 指标 | 正常 | 警告 | 动作 |
|---|---|---|---|
| CTR | >8% | 5-8% | 换封面 |
| 完播率 | >40% | 25-40% | 优化前30秒 |
| 爆点得分 | >30 | 15-30 | 观察/优化 |

## 诊断标准速查

| 维度 | 标杆 | 健康 | 一般 | 差 |
|---|---|---|---|---|
| 赞率 | >3% | 1.5-3% | 1-1.5% | <1% |
| 播放分布 | <30%头部 | 30-50% | 50-60% | >60% |
| 留存（短剧1h） | 1%处>80% | 3分钟>30% | 5分钟>25% | 低于基准 |
| 流量（推荐） | >50% | 30-50% | 20-30% | <20% |
| CTR | >8% | 5-8% | 2.5-5% | <2.5% |

## 阈值出处

以上阈值来自 `diagnosis_engine.py` 的真实运行逻辑，基于追劇姐妹频道基线 [来源:追劇姐妹基线 2026-07]。不同频道可能有差异，诊断时需标注数据来源。
