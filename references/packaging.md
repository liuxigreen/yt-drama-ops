# 包装模式

## 标题公式

```
【冲突钩子】+【身份反差】+【反转词】+【情绪终点】
```

### 句式模板

**伤害→离开→后悔**
先写具体伤害事件，再写主角离开/转身，最后写反派后悔/跪求

**低位→误判→高位揭露**
先给低位身份和被轻视，再制造嘲笑或误认，最后揭露真实身份和碾压结果

**旧命运→重生→新选择**
先说明过去被消耗或死亡，再写重生/觉醒，最后写拒绝旧选择并清算

**全员退缩→唯一敢→全场震惊**
先写所有人退缩/不敢/嘲笑，再写弱者完成不可能任务，最后全场震惊

**偶然相遇→身份隐藏→爱情/继承人结果**
用救人、误进、收留等偶然事件触发隐藏身份关系，最后给甜宠或继承人结果

## 封面×标题协同

### 核心规则
封面和标题围绕同一个"核心钩子"分工协作——标题说清身份反差/情节反转，封面把最有张力的一瞬间视觉化。

### 协同模式

**标题给反差，封面给证据**
标题揭示隐藏身份，封面必须用视觉证据来证明这种身份反差是真实可信的。

**封面定格冲突，标题补因果**
封面只需呈现冲突最高潮、最富张力的一幕（如跪地、拥抱、对峙），标题则负责解释这一幕为何发生以及之后反转为何。

**情绪反转视觉化**
标题描述从冷漠到后悔、从被弃到被宠的情绪转变，封面用人物的表情（冷脸vs甜蜜）、距离（疏远vs亲密）和姿态（站立vs依偎）来可视化这种反转。

**阶层符号强化爽感**
豪车、保镖、直升机 vs 破旧衣服、工装、菜篮——视觉阶级反差。

### 反模式（必须避免）

**题材错位**：标题讲社会逆袭，封面却像奇幻冒险
**只美不钩**：男女主颜值很高，但没有身份符号、冲突动作、关系张力
**标题有爆点，封面无证据**：标题写了cheating、reborn，但封面只有普通合照
**信息过散**：封面中人物过多（>5人）、文字过多、符号堆砌

### 封面要素

| 要素 | 要求 |
|------|------|
| 人物 | 2-4人为主，必须有明确主次关系 |
| 表情 | 必须"外露"：震惊、含泪、愤怒、得意、冷漠 |
| 色彩 | 浪漫用暖金/粉；复仇用红黑/冷蓝金；男频用高饱和金/红 |
| 文字 | 极简，不超过5-7个词，只提炼核心承诺 |
| 道具 | 身份类（西装/工装）、情感类（婚戒/离婚协议）、权力类（王座/龙纹） |

### 性别差异

**女频**：核心是"关系张力"。优先使用2人构图，聚焦男女主。通过肢体距离、眼神、道具来表达爱情、误会、背叛、宠爱等情绪。

**男频**：核心是"低位受辱与力量碾压"。常用1位强主角居中，辅以反派或群众在侧。通过主角的冷静姿态、反派的震惊/跪地、以及战力符号来展示爽点。

## 标签策略

### 四层标签矩阵

1. **题材层**：#ceolovestory #reborn #revenge #drama
2. **情绪层**：#love #betrayal #regrets #beggging
3. **格式层**：#shortdrama #minidrama #FULL #ENG DUB
4. **语言层**：#cdrama #kdrama #engsub #español

### 规则

- 标题标签少而精（1-5个），用于定义核心题材
- 描述标签多而广（10-20个），用于扩展搜索流量
- 定期更新标签库，关注新兴题材标签
- 确保标签与视频内容实质相关

## 描述模板

### 结构
高播放描述遵循"钩子前置，信任构建，流量沉淀"结构。

1. **首两行**（移动端可见）：必须包含剧情钩子或核心承诺
2. **中间部分**：简要补充剧情或频道定位
3. **结尾固定区域**：标签矩阵、订阅引导、社交媒体链接

### 类型

**剧情简介型**：用1-2句话复述并扩展标题的核心冲突和反转

**频道欢迎型**：开头说明频道内容属性，再接剧情简介或标签

**合规声明型**：开头提示"Viewer discretion advised"，再进入剧情简介

**多语言导流型**：强调"FULL"、"Multi SUB"、"PT DUB"等观看属性

### 规则

- 描述前两行是黄金位置，不要被订阅链接占据
- 剧情简介要克制，讲清核心矛盾即可，不要剧透所有反转
- 订阅和引导语放在剧情简介之后
- 标签按语义分组，不要无序堆砌
